Описание задания

  1. Приложение сохранить как отдельный модуль(работающий на локальной машине!) с именем Game_LastName_GameTitle.
  2. Миниавторизация (Ввод имени).
  3. Рейтинг игроков/пользователей.
  4. 3 уровня игры в зависимости от задания (2 только у некоторых). Внутри каждого уровня 3-5 раз генерится похожий вопрос. Например, выбираем животных. На одном уровне просто выбрать животное по какому-то признаку: с четырьмя ногами, с двумя ногами, с шерстью, с перьями, бегает, плавает. Т.е коллекция животных одна, коллекция вопросов одна, и несколько раз выкидываются разные вопросы.
  5. Произвольная генерация вопросов на каждом уровне, случайное расположение элементов игры.
  6. Возможность завершить уровень в любой момент. Переход на следующий уровень осуществляется только при успешном прохождении предыдущего уровня.
  7. При завершении игры есть возможность начать игру с начала (возможно с новым именем).
  8. Минимальное количество страниц 3 (заставка, игра, рейтинг/финал).
  9. Должны быть реализованы различные виды событий 3 (перемещение, двойной клик, наведение, клавиши на клавиатуре . . .) Например, на каком-то уровне выбираем через контекстное меню, где-то тащим по линии, где-то перетаскиваем в нужное место, где-то объекты летят, падают, трясутся, скачут....
  10. Ограничение работы приложения по времени.
  11. Ускорение работы программы, в зависимости от уровня сложности.
  12. Продумать логику игры.
  13. Обязателен подсчет баллов, штрафные балы, изменение баллов от уровня сложности.
  14. Проверка окончания игры (по времени, по баллам, ...). Распределение баллов на усмотрение создающего игру.
  15. Динамическое создание элементов.
  16. Собственные интересные решения (с использованием js).
  17. Допускается использование внешних графических библиотек, которые не влияют работу модуля на локальном, например моем, компьютере).
  18. Данные после каждой игры сохраняются (localstorage, JSON) и передаются на другую страницу, для данного пользователя.
  19. Анимация элементов.
  20. Проект выложен на Ваш сайт на хостинг.
  21. Индивидуальное обсуждение задания для курсовой работы.
  22. Это базовые требования. В остальном, разбудите фантазию)
